Пройтись по git+файловой системе и сверить с документами: STATUS.md, CHANGELOG.md, milestones.md, backlog/. Показать расхождения и предложить патчи одним пакетом.

No arguments. Запускать в конце дня / после серии задач: `/status-review`.

Скилл **только показывает и предлагает**. Ничего не правит до явного «да». Идемпотентный — повторный запуск через минуту ничего нового не находит.

## Шаги

1. **Прочитать шапку `docs/STATUS.md`** — извлечь `commit <hash>` и `Обновлено: YYYY-MM-DD`. Если файла нет — это первая инициализация, перейти сразу к шагу 7 («предложить начальный STATUS.md»).

2. **Собрать дельту с момента шапки**:
   ```
   git log <hash>..HEAD --oneline
   git log <hash>..HEAD --name-only --pretty=format:'%H %s'
   ```
   Сохранить: список коммитов с типами (`feat:` / `fix:` / `perf:` / `docs:` / …), список затронутых путей, упомянутые `#NNN`.

3. **Проверка A — коммиты без отражения в документах**. Для каждого коммита с момента шапки:
   - `feat:` / `fix:` / `perf:` — есть ли строка в `CHANGELOG.md` (в `## [Unreleased]`)? Если нет — кандидат на добавление.
   - Трогает `src/`, `include/`, `fixtures/`, `tests/` — отражено ли в `docs/STATUS.md` («Работает end-to-end» / «Чек-лист v0.1»)?
   - Содержит `(#NNN)` — задача `#NNN` действительно лежит в `backlog/completed/`?

4. **Проверка B — задачи в `backlog/wip/` без движения**. Для каждого файла в `wip/`:
   - Прочитать `**Дата старта:** YYYY-MM-DD`.
   - `git log --since=<дата старта> -- <путь>` + поиск `#NNN` в `git log --since=<дата старта> --oneline`.
   - Если > 7 дней без коммитов, упоминающих `#NNN` — флаг «застряло». Предложение: `/checkpoint` или откат в `new/`.

5. **Проверка C — задачи в `backlog/completed/` без DoD-секций**. Для каждого файла, чей коммит-перемещения в `completed/` попал в дельту шагa 2:
   - Должны присутствовать секции **Как работает**, **Чем управляется**, **С чем связана**, **Диагностика** (см. `backlog/README.md` §«Жизненный цикл задачи», п. 5).
   - Нет хотя бы одной — флаг «не достроено».

6. **Проверка D — dogfood без записи в milestones**. Если в дельте есть коммиты с маркерами прогона (`dogfood`, имена внешних проектов из последних записей `milestones.md`, новые `Прогон N` в commit message body) — есть ли соответствующая запись в `docs/milestones.md` с сегодняшней или близкой датой? Нет — флаг.

7. **Проверка E — STATUS ↔ CHANGELOG**:
   - Каждая фича со статусом ✅ в `## Работает end-to-end` STATUS.md — упомянута хотя бы строкой в `## [Unreleased]` CHANGELOG.md? Нет — флаг.
   - Каждый пункт ✅ в `## Чек-лист v0.1` STATUS.md — то же.
   - **Не предлагать переписывать CHANGELOG задним числом.** Только добавление недостающих строк в Unreleased.

8. **Игнорировать всегда:**
   - `backlog/pending/` — парковка, не очередь (см. memory).
   - Коммиты с типом `docs:` для `backlog/` или `docs/STATUS.md` — это часть процесса, а не контента.
   - `chore(tasks):` коммиты, перемещающие файлы — это уже учёт.

## Отчёт

Сформировать таблицу-сводку и показать пользователю. Шаблон:

```
### Расхождения

| # | Где | Что не отражено | Что предлагаю |
|---|-----|-----------------|---------------|
| 1 | CHANGELOG.md | feat(rules/drift): DRIFT.1/2 (#009 #040) | Добавить строку в Unreleased / Added |
| 2 | STATUS.md   | --baseline (#030) — есть в Чек-листе, нет в Работает end-to-end | Добавить bullet |
| 3 | milestones.md | прогон на duckdb сегодня | Добавить запись «Прогон N — duckdb» |

### Задачи без движения

| # | Файл | Дата старта | Дней без коммитов | Предложение |
|---|------|-------------|-------------------|-------------|

### Завершённые без DoD-секций

| # | Файл | Не хватает | Предложение |
|---|------|------------|-------------|

### Открытые баги корректности (для секции STATUS.md)
- <строка если выявлен>

### Обновить шапку STATUS.md
- было: `commit <старый> · Обновлено: <старая дата>`
- станет: `commit <HEAD> · Обновлено: <сегодня>`
```

Если расхождений нет — короткий вывод: `✅ всё синхронизировано. HEAD совпадает с шапкой STATUS.md.` И всё, ничего не править.

## После «да»

Применить патчи одним пакетом, **без коммита**:

1. Edit `CHANGELOG.md` — точечно дописать в `## [Unreleased]`. Каждый bullet — одно предложение в стиле существующих.
2. Edit `docs/STATUS.md` — обновить релевантные секции + шапку (`Обновлено: <today>`, `commit: <HEAD short>`, `фаза:` если поменялась).
3. Edit `docs/milestones.md` — добавить минимальную запись прогона по шаблону в конце файла (пользователь дополнит детали).
4. Edit задач из «без DoD-секций» — добавить заголовки секций с placeholder-комментарием `<!-- TODO -->` (не выдумывать содержимое).
5. **Коммитить — НЕ из этого скилла.** Пользователь сам решает: `/commit chore(process)` или ручной коммит. В отчёте напомнить: «изменения в worktree, не закоммичено».

## Что НЕ делать

- Не переписывать CHANGELOG задним числом. Только дополнение Unreleased.
- Не двигать файлы в `backlog/`. Это работа `/checkpoint` и `/fix-issue`.
- Не трогать `backlog/pending/`.
- Не запускать билд / тесты / lizard. Это не часть аудита.
- Не править `docs/architecture-spec.md`. Это дизайн, а не статус.
- Не выдумывать содержимое DoD-секций. Только каркас.

## Тон

Сухо, табличками. Без пересказа что было сделано — пользователь и так помнит. Цель — поймать **расхождение между реальностью и документами**, не отрапортовать о работе.
