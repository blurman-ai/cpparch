## Why this exists

#127 closed its **main applicability goal** (supercollider 2929→69, independently re-verified,
no own-code over-exclude) and moved to `completed/`. But three items remained open and were
split out here so #127 could close honestly rather than sit half-done in `wip/`:

1. the **mandatory fixtures** the acceptance bar requires (never written);
2. the **documented residual tails** (bpftrace 3 FPs, newsboat unchanged);
3. the **#129 path-trio gap** — 4 default rules still do not exclude generated files.

See #127 (`backlog/completed/127_maj_vendor_generated_exclusion.md`) for the full evidence,
the layer design, the over-exclude bug that was caught, and why `.gitmodules` / copyright-mismatch
were demoted.

## 1. Fixtures (mandatory — the unmet acceptance bar)

CLAUDE.md / MVP.md: *"if a feature cannot be tested with fixtures — do not implement it."* The
shipped exclusion (the `external_libraries` token, `isGeneratedPath += .tab.h`, `hasGeneratedHeader`
with `@generated` / `coco/r` / `linux-syscall-note`, `hasMinifiedContent`) has unit coverage but
no fixture dirs. Add them:

- [ ] `fixtures/vendor_exclusion/pass_nested_license/` — `libs/foo/LICENSE` + code excluded, root LICENSE not.
- [ ] `fixtures/vendor_exclusion/fail_own_code/` — own code with a real violation NOT excluded (anti-over-exclude).
- [ ] `fixtures/vendor_exclusion/pass_generated_marker/` — a file with `// @generated` excluded.
- [ ] `fixtures/vendor_exclusion/pass_minified_data/` — an avg-line>110 data blob excluded from all scans (#147).
- [ ] (if the `.gitmodules` layer is ever added) `fixtures/vendor_exclusion/pass_gitmodules/` — a `.gitmodules` path excluded.

Anti-over-exclude regressions must include the exact prose strings that previously ate own code
(`bytecode generated by bpftrace`, `imported from pre-compiled BPF programs`) — these are already
in the unit test; mirror the intent at fixture level.

## 2. Residual vendor/generated tails (calibrated layer, do not force)

From #127's enumeration — real compiled-in/copied code not covered by **safe** name-independent
signals:

- **bpftrace SF.8 ×3** — `bpf_assembler.h` (`copied from bcc`; the `copied from` marker was dropped
  in #131 as broad-corpus-risky, confirmed own-FP on libjxl), `compiler-clang.h` / `errno.h` (bare
  single-line `SPDX GPL-2.0`, no distinguishable marker, no nested LICENSE). Candidate signals,
  each needs corpus calibration + enumeration of non-project files before shipping:
  a `src/stdlib/include` style path token, or SPDX-owner-mismatch ONLY in conjunction with another
  signal (never alone — #127 reverted copyright-mismatch-alone for eating mne-cpp / micropython).
- **newsboat GodHeader/SF.* ×?** — `filter/Parser.h` / `Scanner.h` (Coco/R) carry no text banner in
  this version, only the include guard `COCO_PARSER_H__` / `COCO_SCANNER_H__`. The guard `coco_*_h__`
  is a near-zero-FP signal but overfits one tool version → only add on request / with calibration.

Acceptance for this part is **honest residual accounting**, not zero: each tail either covered by a
calibrated+enumerated signal, or explicitly left documented (no silent forcing).

## 3. The #129 path-trio gap (behavioural — needs fixtures + its own commit)

#129/#127 found 4 default rules that classify **content-free git paths** and never went through
`AuthoredScope` / `SourceSnapshot`, so they do **not** exclude generated files:
`god_file_growth`, `defect_attractor`, `satd`, `test_co_evolution`. For `god_file_growth` both
call-sites (`calculateP75`, `detect`) even carried the comment "exclude vendor, test, **and
generated**" while the code did only vendor-dir / test checks (the comments were corrected; the gap
remains).

- [ ] Add `scan::isGeneratedPath` (content-free: `_wrap.cpp` / `.pb.h` / `moc_` / `.tab.*` …) — and
      likely `isVendoredBasename` — to the shared path predicate of these 4 rules.
- [ ] This changes default-rule behaviour on repos with generated code → fixtures + a **separate**
      commit, never a silent edit.

## Acceptance

- [ ] The 4 (+1 conditional) fixtures exist and pass; `fail_own_code` proves no over-exclude.
- [ ] bpftrace / newsboat tails: each covered by a calibrated+enumerated signal OR documented as a
      deliberate residual (no forcing, no single-line-SPDX, no `copied from`).
- [ ] The 4 path-trio rules exclude generated paths, behind fixtures, in their own commit.
- [ ] Dogfood: `archcheck src/ include/ tests/` stays 0 (no own-code over-exclude).
- [ ] Transparency line `excluded N files as vendored/generated (reason: …)` — wire the single
      mechanism across all reasons (deferred from #127).

## Do not do

- Don't ship copyright/SPDX-owner mismatch as a **sole** signal (reverted in #127: ate mne-cpp,
  micropython own code).
- Don't add `copied from` to the marker set (broad prose surface, own-FP on libjxl — #131).
- Don't overfit one generator's include guard without a calibration request.
- Don't re-open the applicability win — supercollider/the token layer is shipped and verified.
