## Цель

Перестать выдавать ложный позитив `Lakos.GodHeader` на precompiled header файлах (`pch.h`, `stdafx.h` и т.п.), которые по определению включаются во все единицы трансляции.

## Контекст

При прогоне archcheck на реальных проектах (`gm/rmi`, `samastra_itain`) правило `Lakos.GodHeader` срабатывает на `pch.h` / `rmi/pch.h` с fan-in 60–111. Это ложный позитив: precompiled header намеренно включается в каждый `.cpp` — это паттерн, а не нарушение.

Пример ложного вывода:
```
rmi/pch.h: [Lakos.GodHeader] fan-in 111 exceeds threshold 30
```

## План выполнения

- [ ] Добавить хардкод-список известных PCH-имён: `pch.h`, `stdafx.h`, `precompiled.h`, `precompiled_header.h`
- [ ] В правиле `GodHeader` пропускать файлы, чьё имя (без пути) входит в этот список
- [ ] Добавить поддержку пользовательских исключений через `.archcheck.yml`: `godheader_exclude: [my_pch.h]`
- [ ] Добавить fixture: `fixtures/lakos_godheader/pass/pch_excluded/` с `pch.h` и fan-in > 30
- [ ] Обновить тест `GodHeader` — проверить, что `pch.h` не выдаёт нарушение

## Ключевые решения

| Решение | Причина |
|---------|---------|
| Хардкод имён + конфиг override | Хардкод покрывает 95% случаев без единой строчки конфига; конфиг нужен для нестандартных имён |
| Фильтр по имени файла, не по пути | PCH может лежать в любом месте дерева |
