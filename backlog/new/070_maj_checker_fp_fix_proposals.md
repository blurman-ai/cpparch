# [SCAN] Фиксы FP дубликат-чекера по итогам глазной верификации (#067)

**Дата создания:** 2026-06-02
**Дата старта:** —
**Статус:** new
**Модуль:** SCAN / rules (duplication #053/#056/#059)
**Приоритет:** major
**Related:** #060 (валидация/харденинг чекера), #067 (глазная верификация), #056 (token-проход)

## Цель

Разобрать и реализовать предложения по снижению false positives дубликат-чекера,
собранные агентами при глазной верификации 135 drift-реп (#067). Это рабочий
backlog фиксов — **разбираем потом**, приоритизируем по числу случаев на класс.

## Контекст (метрики верификации #067, 2026-06-02)

- Проверено агентами: **135 drift-реп** (глазами, по реально прочитанному коду).
- Подтверждённый копипаст: **87/135 реп**, **195 случаев**.
- Ложных: **406** → **precision ≈ 32%**.
- Подтверждённых include-циклов: **21** (в авторском src, не вендоре).
- Классы FP: idiom=189, coincidental=99, whole-file=51, other=31, data-table=21,
  generated=9, header-impl=4, boilerplate=2.

Полный разбор (каждый FP: что обмануло детектор + где) — в
`experiments/ai_repo_run/VERIFICATION_REPORT.md`.

## Как разбирать

Фиксы ниже сгруппированы по классу FP и отсортированы по объёму проблемы.
Первый приоритет — **idiom (189)** и **coincidental (99)**: вместе ~71% всех FP.
Многие фиксы повторяют одну идею (стоп-листы фреймворк-бойлерплейта, down-weight
логирующих макросов, AND-gating по contiguous-run, file-local IDF) — при разборе
схлопнуть в несколько общих механизмов, а не реализовывать 405 правил.

## Предложения по фиксам (из #067, как есть — дедуп при разборе)

### Сгруппированные предложения по фиксам (по классам FP)

**idiom** (189 случаев):
- фикс: Эвристика: блоки, состоящие преимущественно из ostream-цепочек логирования ('<<' chains) и однотипных do_call/bind-строк, маркировать как low-signal; требовать минимум N уникальных не-словарных идентификаторов на окно прежде чем засчитать near-пару.
- фикс: Добавить в #056 нормализацию/деунификацию error-handling guard-боллерплейта RPC: распознать паттерн 'er.code=...; er.message=...; return false;' как шумовой и не учитывать в line-similarity знаменателе.
- фикс: Down-weight tokens belonging to project-wide logging/assert macros (LL_WARNS/LL_INFOS/LL_DEBUGS/LL_ENDL) before computing token similarity, and gate reporting on line-metric >= ~0.6 rather than the inflated token weight w.
- фикс: Add an LLView/event-handler boilerplate stop-list (handled flag pattern, getWindow()/setCursor, hasMouseCapture/setMouseCapture) to selective normalization, or require that the differing 'meat' tokens (non-boilerplate identifiers) exceed a minimum count before a within-file pair is emitted.
- фикс: Treat large switch ladders over platform enum constants (SUBLANG_*/LANG_*) as low-entropy: weight each 'case CONST:' lower, and require non-case statement overlap, so two unrelated Win32 switch blocks don't pair.
- фикс: Тот же фикс, что и для mesh.hpp: callback-registration idiom ([cb](Variant&...){ ... return false; }) внести в авто-детект шаблонных идиом — если совпадение целиком объясняется одним повторяющимся lambda-скелетом, схлопывать в одну канонич. форму и не репортить попарно.
- фикс: #056: maintain a Qt-UI-idiom stop-pattern filter — runs of new Q*Layout/new Q*Button/addWidget/addStretch/addRow/connect(...accept|reject) are framework scaffolding, not logic. Either strip these lines before hashing (like comment-stripping) or require that a flagged chunk contain >=N non-Qt-API statements (real control flow / domain calls) to count.
- фикс: #056: treat QMetaObject::invokeMethod / std::thread([..]) capture-list + lambda-body posting as a recognized concurrency-dispatch idiom; the variable capture list inflates token overlap. Normalize capture lists out of the token stream, and again require AND-gating with contiguous line run.
- фикс: Добавить в #056 allowlist коротких well-known идиом по сигнатуре вызовов: блоки <=10 строк, состоящие преимущественно из вызовов известного crash/signal API (backtrace/backtrace_symbols_fd/signal/raise/abort), не считать дублем. Либо общий порог: verbatim-блок <12 строк, где >60% токенов — имена внешних C-API функций, понижать в severity.
- фикс: Add an errno/strto* error-handling idiom to the #056 idiom-suppression list (like the getopt/switch-dispatch idioms): a block dominated by errno==ERANGE / endptr=='\0' / overflow!=NULL guard tokens should be down-weighted, since it is a standard C numeric-parse epilogue copied by convention, not logic reuse.
- фикс: Detect the visitor-method shape (function whose body is dominated by `symbol->accept(*this)` / `return NULL;` / a single `dynamic_cast<*_c>` dispatch) and raise the min-similarity threshold for such methods, or require a longer matched run (e.g. >=8 non-idiom tokens differing) before flagging within-class visitor pairs. Otherwise any AST-visitor compiler self-reports as massively duplicated.
- фикс: For headers, strip Qt-meta keyword runs before hashing: collapse Q_PROPERTY/Q_OBJECT/Q_SLOTS/Q_SIGNALS/Q_EMIT and the READ/WRITE/NOTIFY/RESET property-keyword tokens into a single sentinel so the surviving fingerprint is the property/slot NAMES, not the macro grammar. Equivalently, add a header-class-skeleton suppressor: if matched span is >70% macro/access-specifier/declaration tokens and contains no statement bodies, downweight. This keeps real impl clones (the .C apply()) while killing parallel-class-declaration noise.
- фикс: Strip comments before tokenizing (#056 should already normalize to code tokens). Verify the token pass does not include comment text or box-drawing/section-marker lines; if it does, drop comment tokens entirely so the clone score reflects executable structure, not shared documentation scaffolding.
- фикс: Понижать вес токенов, образующих repeated single-line dispatch-идиому: если строки региона совпадают по шаблону `else if (id == LIT) { CALL }`, считать его idiom-table и не учитывать как chunk-копию (тот же фильтр data-table, расширенный на guard-ladder).
- фикс: Требовать минимальное совпадение по идентификаторам-вызовам (а не только по ключевым словам): если пересечение множеств вызываемых функций между ADDED и BASE < ~30%, отбрасывать пару даже при высоком token-overlap.
- фикс: #056: добавить эвристику 'struct-init boilerplate suppression' — если блок после нормализации на >70% состоит из паттерна <decl T x{}> + последовательности <x.field = CONST/литерал> без управляющих конструкций (нет if/for/call с аргументами-выражениями), снижать вес: это декларативная инициализация API-структур, а не алгоритм. Дополнительно: домен-стоп-лист префиксов токенов (VK_STRUCTURE_TYPE_, GL_, D3D_) — высокая доля таких констант в окне => штраф к w.
- фикс: #056: при cross-file сравнении header-блоков, если совпадающий кусок состоит преимущественно из объявлений virtual-методов наследников ОДНОГО базового класса (`: public Base` + серия `virtual ... override/=0`), понижать вес — это реализация интерфейс-контракта (OCP plugin family), не дрейф. Эвристика: если >60% совпавших строк — `virtual`-объявления и оба класса делят один base-symbol, классифицировать как 'interface-impl' и не флагать.
- фикс: Add a same-file structural-distinct guard: when both windows live in one .cpp, require shared n-gram contiguity (e.g. >=6 consecutive matching normalized tokens spanning >50% of the smaller window), not just bag-of-tokens weight. Also detect the closed-enum dispatch idiom: if both windows are dominated by 'case <Enum>::<ID>:' / 'if(id==<const>)' labels that differ only by the enum-member/const token, down-weight as enumeration boilerplate rather than copy-edit. A file-local IDF (penalize identifiers that recur >K times across the whole file, like _xApp/Mode::APP_*) would also collapse this w=0.92 toward noise.
- фикс: Добавить domain-стоп-лист коротких библиотечных teardown/init-идиом (libbpf: bpf_link__destroy цикл + bpf_object__close; setrlimit RLIMIT_MEMLOCK), либо повысить минимальную длину уникальных токенов: блоки <12 строк, состоящие преимущественно из вызовов внешнего API без проектной логики, не считать копипастом. Совместно с move-фильтром из предыдущего пункта оба хита уходят.
- фикс: #056: при within-file near (line>0.9) требовать совпадение не только токенов, но и control-flow скелета (loop/branch shape) — если CFG-фингерпринты блоков расходятся (heap-loop vs linear), понижать вес. Альтернатива: исключать совпадения, целиком состоящие из вызовов одного протокол-API (Crc32::compute, memcpy, sizeof(FrameHeader)) без присваивания доменных полей.
- фикс: #056: эвристика 'mirror pair' — если в паре один блок доминируют write_*-идентификаторы, а другой read_*/parse_* (или префикс write vs read у вызовов лямбд), классифицировать как ser/deser-зеркало и подавлять. Сигнал: высокая доля идентификаторов, отличающихся только токеном write<->read при общем хвосте поля.
- фикс: #056: подавлять within-file пары, где BASE-блок — определение лямбды/хелпера, а ADDED — его вызовы (или наоборот): они структурно делят имена параметров. Детект: пересечение токенов сосредоточено в именах локальных лямбд (read_u32/write_u8), объявленных в той же функции.
- фикс: Не репортить пары, обе стороны которых лежат внутри ОДНОЙ функции (один и тот же enclosing symbol) — это внутрифункциональное повторение идиомы, а не клон куска. Доп. эвристика: если общий префикс/суффикс матча — это распознанный 'event-dispatch' паттерн (короткий hb_vmPush*/hb_vmSend цепочка <= ~8 строк), штрафовать вес; поднять минимальную длину уникального (не-идиомного) ядра матча, исключив из подсчёта токенов известные VM-bridge вызовы.
- фикс: #056: trim общих ведущих guard-строк (leading-guard stripping) перед сравнением окна, либо требовать, чтобы совпадающий регион имел >= K не-guard строк в теле; снизить вес окон, чьё совпадение целиком в первых N строках функции.
- фикс: #056: библиотечно-идиоматические токены (WSAGetLastError, sockaddr_in, ::socket/::connect/::accept, INVALID_SOCKET) держать в domain-idiom stop-list; пара из <=2 различающихся идентификаторов на фоне platform-API вызовов -> downweight.
- фикс: Project-specific stopword/idf weighting: down-weight tokens that occur in >X% of windows of the same file (template_param*, std::nullopt, gTemplateRegistry, FLASH_LOG*). A window whose matched tokens are dominated by high-frequency project identifiers should not reach w=1.00. Require a minimum count of LOW-frequency (rare) shared tokens before confirming.
- фикс: Treat the visitor dispatch skeleton (is<...>()/as<...>() pairs and the wrapped type-name) as a single normalized token that carries the TYPE ARGUMENT. Two windows that share the skeleton but differ in the <NodeType> arguments (SehTryExcept vs SwitchStatement vs DeclarationNode) should diverge sharply. Currently normalization erases the type args, making all visitor blocks look identical.
- фикс: Same idf remedy: maintain a per-subsystem high-frequency identifier table (IrGenerator: sema_normalized_current_function_, TypeCategory, InternalError, ExprResult). Matches whose shared n-grams are >70% drawn from that table get their weight discounted so they fall below the report threshold.
- фикс: Add a control-flow-shape gate: require structural similarity beyond the token bag — e.g. matching sequence of branch/loop keywords and call targets. The two blocks differ in their distinct call set (push_back/setCategory vs lookupTemplate/rfind), so a Jaccard on CALL-TARGET identifiers (not all identifiers) would separate them.
- фикс: Ввести жёсткий gate по line-similarity для within-file пар: при line<0.30 пара не считается chunk-копией даже если token-weight высок (токены совпадают из-за общего словаря, а не из-за структуры). Альтернатива — стоп-лист идиом popen/system-обёрток: блоки, где >60% значимых токенов это {system,cmd,c_str,cout,cerr,return,package}, понижать в весе как boilerplate.
- фикс: Downweight blocks that are pure switch/dispatch: if a candidate chunk is >60% case-label + single-call-per-case lines (a jump table), require a higher w threshold (e.g. >0.97) AND that the called function names differ in more than a fixed suffix (_eval vs _pdf). A dispatch shim where bodies differ only by an eval/pdf suffix is intended polymorphism, not drift.
- фикс: Down-weight spans dominated by a single recurring member-access n-gram (here m_fixedParamSetsCompPtr/m_paramSets index split) that appears >2x in the same file; require a minimum count of distinct statements (not just token length) before flagging within-file.
- фикс: Raise the minimum token/line floor for within-file pairs to ~10 effective lines, and exclude pairs where one body is a pure forwarder (single assignment + single return/call) — overload sets trip this constantly.
- фикс: For low line-agreement diverged hits (line<=0.6) that are short and adjacent within the same function, suppress: a true copied chunk usually keeps high line-order agreement. Treat low line + short span + same-function as 'parallel branch' and drop.
- фикс: Exclude files named main.cpp / matching '^\s*int main\s*\(' from cross-file dup reporting (or hard down-weight), analogous to the vendored-file exclude already added in #069. Entry points are a known boilerplate class.
- фикс: Treat T-prefixed / template-class headers (and generally header-defined method bodies whose enclosing class is a template) as a separate bucket with a higher similarity threshold before flagging, since shared template scaffolding is expected; report only if divergence pattern looks like edited-copy rather than parallel template.
- фикс: Для within/cross-file пар с высоким token-w но низким line-sim (line < ~0.30) требовать минимальное число СОВПАДАЮЩИХ line-shingle, а не только token-multiset overlap: добавить вторичный фильтр на долю общих 3-строчных shingle (>= 0.2), иначе отбрасывать как 'shared-vocabulary'. Плюс не сравнивать блоки, чьи опорные API-вызовы непересекающиеся (здесь memory.Data[]/FetchWordFromMemory против ioDevices/handlesRead) — anchor-token Jaccard по library-call идентификаторам < 0.15 => drop.
- фикс: Raise the minimum distinct-statement count for a clone: a window whose bodies are all <=1 statement (trivial accessor/forwarder) should not count even when several are adjacent. Filter spans where the median function-body length inside the window is <=1 statement, or require >=K non-trivial statements total.
- фикс: Поднять порог cross-file для коротких 'парсерных' блоков с насыщенным доменным словарём (hex/strtoul/isxdigit/wildcard) ИЛИ требовать совпадения сигнатуры/возвращаемого типа функции-хоста: если ADDED и BASE принадлежат функциям с разным return-type и разным именем символа — понижать вес w на штраф. Также: дедуплицировать выдачу — если у ADDED уже есть within-file близнец с более высоким w, не плодить дополнительный cross-file хит на ту же идиому.
- фикс: #056: add a Qt-connect-run filter — collapse/skip windows whose token stream is dominated (>~70%) by a repeated 'connect ( ... )' call shape before scoring; or down-weight blocks where the same call-callee token (connect/QObject::connect) repeats N+ times, since signal-slot wiring is structural, not duplicated logic.
- фикс: #056: detect intra-block N-fold parallel repetition (>=4 sibling if/branches differing only by one enum/identifier token) and normalize them to a single representative before cross-block comparison, so per-sensor/per-field fan-out stops inflating self-similarity.
- фикс: Down-weight or strip full-line comment tokens before shingling for within-file matches (a project that comments every statement inflates token overlap). Additionally, require an identifier-skeleton match (sequence of called functions / control-flow keywords with comments removed) above a threshold before reporting; w(token)>=line-similarity gap (here line=0.89 vs token=1.00 — a big token-vs-line gap is a comment-driven idiom signature) should trigger suppression.
- фикс: Raise the within-file reporting floor for the 'diverged' band (w<0.80 AND line<0.75) so heavy-comment LLP-style files stop producing noise; or gate diverged matches on a minimum count of shared *non-boilerplate* tokens (exclude project-wide hot identifiers like debug::log, FUNCTION, the connection ptr name) computed from a per-repo idf.
- фикс: Same idf-based hot-token discounting as above. The line-similarity here is only 0.50 while token w=0.70; a rule 'suppress when line<=0.55 for within-file' would drop this and the miner.cpp/data.cpp(627) cases without losing the real 80ebc04 cross-file hit (which has high line similarity).
- фикс: Enforce a minimum substantive window size (>= ~15 non-comment, non-brace logical lines) before reporting a within-file pair; tiny windows dominated by comments/logging are almost always idiomatic FPs in this codebase.
- фикс: Enforce a minimum matched-line floor (>=6-8 non-trivial lines) before reporting, and exclude pure-punctuation/brace lines from the line-similarity denominator so 4-line brace-heavy windows can't reach high w.
- фикс: Make the chunker function-aware: do not let a candidate window straddle a function boundary, and discount windows dominated by a single repeated guard idiom (if(!call(arg)) return false;) sharing one argument identifier.
- фикс: Игнорировать пары, где оба куска присутствовали ДО проверяемого коммита (blame-gate): сравнивать только токены, реально добавленные/изменённые в diff коммита, а не весь файл, оказавшийся в дереве merge-коммита. Это убрало бы все 12 хитов разом.
- фикс: Порог по абсолютной длине куска: для блоков < ~6 непустых строк требовать w>=0.97 И line>=0.9; короткие STL-обёртки иначе всегда дают ложный 'near'. Плюс не считать совпадением, если >70% токенов блока — это вызовы из одного namespace std::/boost:: (общий внешний словарь).
- фикс: Эвристика 'overload family': если два куска в одном файле имеют общий префикс имени функции (log/logf/logv) и оба содержат va_start/vsnprintf — понижать score, это ожидаемое дублирование вариадик-обёрток, а не дрейф.
- фикс: #056: maintain a small allowlist of canonical micro-idioms (trim, split, ltrim/rtrim, toLower) keyed on normalized token-shape + length<=8 lines; suppress a pair when BOTH ends match an allowlisted idiom signature. Keep stripComments (longer, distinctive control flow) un-allowlisted so it still fires.
- фикс: No change needed - this confirms the current chunk-length threshold is correctly excluding short file-IO idioms. If anything, guard against future over-detection: keep a small stop-pattern list for ubiquitous stream boilerplate (ofstream open/is_open/close, istreambuf_iterator content reads) so the open/guard prologue alone never anchors a hit.
- фикс: Add a test-harness boilerplate filter: drop tokens belonging to recognized harness lines (std::cout<<"Test:, assert(, checkmark Passed, push_back/static_cast scaffolding) before shingling, OR weight down shingles that consist mostly of stream-IO + assert tokens. Require the matched window to contain a minimum count of DISTINCT non-IO/non-assert statement tokens (e.g. real call sites / operators) before reporting.
- фикс: Framework-idiom downweighting: maintain a small list of high-frequency Qt API call tokens (new QAction, connect, &QAction::triggered, addAction, setStatusTip, setStyleSheet, tr) and exclude windows where >~70% of statements are such single-call framework registrations. Equivalently, require a minimum count of control-flow / non-trivial-expression statements in the window before reporting a UI/dispatch block.
- фикс: Same Qt-idiom downweighting as above plus a guard: do not pair two windows that straddle a function boundary where each window is itself a short (<~12 stmt) event handler dominated by widget-accessor + QMessageBox calls. Add function-boundary awareness so the matched window is anchored to a single function body rather than a sliding range that spans the tail of one slot and the head of the next.
- фикс: Add a per-file 'boilerplate-line' suppression: when >40% of the matched lines in BOTH chunks also occur in >=N other functions of the same file (a repeated structural template), down-weight the pair. Concretely for #056: compute line-frequency within the file and exclude lines whose normalized form appears >=3 times file-wide from the token window before scoring (idiom/template stripping), so trampoline scaffolding stops anchoring matches.
- фикс: Detect 'sibling branches of one switch/if-else-chain in a parse loop' as a class: when ADDED and BASE are both children of the same enclosing loop/switch in the same function, raise the threshold sharply (these are expected parallel branches, not extracted-then-edited copies). Requires brace-depth/parent-node info in #056 windows.
- фикс: #056: demote/suppress matches whose token-IoU is high but line-similarity (line=) is below ~0.65 — i.e. require both a token AND a structural-line agreement floor before reporting within-file. Additionally add a settings-serializer idiom filter: blocks dominated by repeated `setValue(...)`/`value(...)` key-value lines are config marshalling, not logic copies.
- фикс: #056: raise the minimum window length for within-file reports (these are ~8 lines) and/or down-weight matches between two adjacent same-purpose helper functions in the same translation unit where the shared tokens are the function's own parameters — small sibling helpers over a common struct are expected, not copy drift.
- фикс: Add a switch-arm-aware filter: if a candidate duplicate block is bounded by `case <Enum>::<Id>:` ... `break;` and differs only in the enum constant + a small set of string/field identifiers, downweight or drop it (this is exhaustive-enum boilerplate). Heuristic: require >=2 statements that are NOT pure assignments-from-renamed-symbols before counting a case body as a real clone; raise the min-token threshold for blocks whose first token is `case`.
- фикс: Маскировать содержимое препроцессорных debug/assert-блоков (#ifdef *DEBUG/*HARDENED/*ABORTIFERROR + тело cerr/abort/log-макросов) перед токенизацией, либо понижать вес токенов из таких блоков. Порог line-similarity ~0.91 на 25-строчных фрагментах, где >60% строк — log/assert-бойлерплейт, не должен триггерить.
- фикс: #056: maintain a small idiom-fingerprint denylist for well-known framework init boilerplate (RCCHECK-style ret-check macros + the micro-ROS/ESP-IDF rcl* init chain), or down-weight chunks whose tokens are dominated by external-API call identifiers (rcl_/rclc_/rmw_/esp_) so SDK setup sequences don't accrue copy-paste weight on their own.
- фикс: Тот же case-table детектор: GL-классификаторы из одних 'case GL_*: return N;' — known idiom; требовать совпадения конкретных enum-меток (а не только формы) и function-aligned окон, чтобы не клеить хвост+голову соседних функций.
