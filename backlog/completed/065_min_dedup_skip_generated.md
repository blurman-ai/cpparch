# [SCAN] Deduplicator: skip in-tree generated code by header marker

**Date created:** 2026-06-01
**Date started:** 2026-06-01
**Date completed:** 2026-06-01
**Status:** done
**Module:** SCAN
**Priority:** minor
**Related:** #056 (token-dup), #064 (vendor-exclude)

## Goal

Remove false copy-paste from in-tree generated code (protobuf `*.pb.cc`, Qt moc,
flatbuffers, ggml template-instances) that vendor-exclude (#064) does not catch —
it lies not in `third_party/`, but in the project tree (`gen/`, next to the code).

## How it works

`hasGeneratedMarker(src)` in `experiments/partial_duplication/main.cpp`: takes the first
~1200 bytes of the file (header ≈ 15–20 lines), lowercases it, looks for generator markers:
`@generated`, `do not edit`, `do not modify`, `code generated by`,
`automatically generated`, `machine generated`, `qt meta object compiler`,
`generated by the protocol buffer`. Found → the whole file is skipped.

The guard sits at a **single** point `collectFromSource` (snapshot/baseline/diff are funneled
here — DRY was achieved along the way: snapshot used to inline the collection bypassing it). On by default;
`--no-skip-generated` disables it.

## How it is controlled

- `Options::skipGenerated = true` (default), flag `--no-skip-generated`.

## Verified

- `Ball-Lang/ball` → `ball.pb.cc` (18410 lines of protobuf): default **0 pairs**,
  `--no-skip-generated` — **26 pairs** (false copy-paste suppressed).
- Calibration: 50/4443 files in the sample marked generated — all genuinely generated
  (protobuf, CoreML whisper-impl, ggml `template-instances/fattn-*`). Hand-written code
  untouched (M65Compiler still finds its 26 pairs).
- build OK (GCC8, `-std=c++2a -lstdc++fs`), lizard clean (function ≤30 lines).
