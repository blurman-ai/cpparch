## Goal

archcheck must **not analyze compiled-in third-party code** (vendored deps) and
**generated files** — otherwise on any real C++ project with bundled dependencies
the result drowns in foreign noise. The current filter (`kVendoredExact` by basename +
a few paths in `file_classification.h`) does not cover the real cases.

## Evidence (goodfirstissue dogfood, 2026-06-14)

Running archcheck over C++ repos from goodfirstissue exposed the problem on 3 of 8 repos,
with varying magnitude:

| Repo | What | Effect |
|---|---|---|
| **supercollider** | `external_libraries/boost/` (4320 .hpp of compiled-in Boost) + ICU | **~2900 violations, >95% — Boost/ICU FP**; real TPs (QtCollider) drowned |
| **bpftrace** | `src/stdlib/include/` (kernel UAPI headers, SPDX GPL-2.0) + `bpf_assembler.h` ("copied from bcc") | SF.8 ×4 FP |
| **newsboat** | `filter/Parser.h` + `Scanner.h` (generated by Coco/R) | SF.8 ×2 FP |

The supercollider verdict is a verdict on applicability: **without a filter, archcheck on a
project with bundled deps is unusable** (5-10 real findings vs thousands of vendor noise).

## Why a naive path filter does NOT solve it (corpus measurement)

A run over the corpus `~/oss` (1686 repos) for "hidden" vendors — directories with
a nested LICENSE/COPYING below the repo root:

- **2183** nested LICENSEs total → **945** caught by tokens
  (`third_party|vendor|external|3rdparty|deps|contrib|...`), but **1238 (57%) — NOT**.
- After filtering out noise (license collections, docs, debian) — **515 directories-with-code-and-LICENSE
  outside the tokens, in 168 repos**: real compiled-in libs in unpredictable paths.
- Identified by name (a naive token filter would miss them): **imgui, eigen-3.3.7,
  Eigen, CMSIS, hidapi, zlib, STM32H7xx_HAL_Driver, miniz, ggml, fmt, tbb, glfw,
  googletest-src**.

That is, vendor-directory names are unpredictable (`external_libraries`, `src/stdlib/include`,
`libs`, the lib's own name) — **we need signals that do not depend on the directory name.**

## How to filter — layers (cheap → expensive, early exit)

**1. `.gitmodules` — almost free and precise.**
A submodule is a third-party repository by definition. Parse `.gitmodules`, exclude
the listed paths. A large class for free.

**2. Nested `LICENSE`/`COPYING` deep in the tree + code nearby.**
For your own project the license is at the root. Your LICENSE in `external_libraries/boost/`,
`src/stdlib/include/.../COPYING` → a compiled-in separate project. Cheap (stat).
IMPORTANT (from the measurement): the qualifier "+ the directory contains .h/.cpp" is mandatory — otherwise
it catches license collections/docs/debian (noise: 1238 → 515 after filtering by code).

**3. Path tokens** — `third_party|3rdparty|vendor|vendored|external|extern|deps|
contrib|submodule|subproject|/ext/|/import/` — as one of the layers (catches ~43% of
nested licenses), not the only one.

**4. Copyright/SPDX mismatch with the repo mode** (reads headers).
Compute the copyright mode across the repo; files with a FOREIGN copyright/SPDX → vendored. Generalizes
"copied from" without an explicit folder (bpftrace UAPI = GPL-2.0 Linux in a BSD repo).
+ header markers: `copied from`, `imported from`, `DO NOT EDIT`, `@generated`,
`autogenerated`, `// Generated by` (also catches the generated class).

**5. Graph island (structural, the graph is already built).**
A vendored lib = a dense subgraph that the project INCLUDES into but which does NOT include back into
the project (directional asymmetry). For a candidate directory:
incoming(project→subtree) > 0, outgoing(subtree→project) == 0, the subgraph is large/
connected → vendor. `external_libraries/boost/` in supercollider = a pure sink.
Does not read content, but needs a threshold; heavier than the rest.

**Recommendation:** layers with early exit: `.gitmodules` → tokens → nested
LICENSE+code → (for the rest) copyright-mode / graph-island.

## Generated files (adjacent sub-class)

Separate from vendor, but solved by the same header markers (`@generated`, `DO NOT EDIT`,
`Generated by`) + known pairs/patterns: Coco/R `Parser.h`/`Scanner.h` (newsboat),
`*.pb.h/.pb.cc` (protobuf), `moc_*`/`ui_*`/`qrc_*` (Qt), `*.tab.{h,c}`/`*.yy.{h,c}`
(bison/flex), `*_generated.h` (flatbuffers).

## Non-migrated rules still do not exclude generated (2026-06-19)

A review of the #129 architecture found: 4 rules on the "path-trio" (god_file_growth,
defect_attractor, satd, test_co_evolution) classify content-free git paths and
did NOT go through `AuthoredScope`/`SourceSnapshot`. For `god_file_growth` both call-sites
(`calculateP75`, `detect`) carried the comment "exclude vendor, test, **and
generated**", but the code did only `pathHasVendoredDir || pathHasTestDir ||
isTestBasename` — neither generated, nor `isVendoredBasename`. The comments were brought
to the truth (docs/comment commit), but **the gap itself remains**: `isGeneratedPath` is
content-free (paths `_wrap.cpp`/`.pb.h`/`moc_`…), it fits these rules without
reading content. When the #127 path layer lands — add `isGeneratedPath` (and,
probably, `isVendoredBasename`) to the shared path predicate of these 4 rules. This is a
behavioral change of default rules → needs fixtures + a separate commit, not a
silent edit.

## ⚠️ Transparency is mandatory ("no silent caps" principle)

Print `excluded N files as vendored/generated (reason: gitmodules / nested-LICENSE
/ copyright-mismatch / island)`. A silent exclusion will one day eat real project code
and we won't notice — exactly the skeptic-risk that has already come up. A flag
`--no-vendor-exclude` for debugging is possible.

## Attempt at copyright-owner mismatch and why it was reverted (2026-06-19)

Tried the cheapest copyright-mismatch layer on top of #129: extract from the header
the "first copyright-owner token" (`copyrightOwnerToken`), take the repo's dominant owner
(>50%), exclude files with a different owner. On rmm this worked (rapidcsv
Berggren ≠ NVIDIA-dominant → clone+cplx 1/3→0/0). BUT **a 200-commit check caught
over-exclusion** — reverted:

- **mne-cpp** (BSD-self-licensed): copyright is by **author names** ("Copyright (C) 2014,
  Christoph Dinh, Lorenz Esch" — different per file) → no dominant owner → the fallback
  excluded its own full-BSD-banner files → cplx 4→0, edges 3→0. Zeroing out
  own code.
- **micropython** `ports/alif` (OpenMV LLC — a contributor port in the tree) +
  heterogeneous copyright in `ports/` → edges 4→0.

**Conclusion:** "different copyright owner" ≠ "vendor". Contributors, ports, co-authors, absorbed
code have a different copyright, but it is the project's **own** code. A naive owner-mismatch
silently loses recall — for an architecture checker that is the worst class of error. **The layer requires
calibration on the corpus** (exactly the list above: `.gitmodules` → nested-LICENSE+code →
copyright-mismatch ONLY together with other signals, not alone → graph-island). Do not
ship copyright-mismatch as the only signal. #129 (dominant-banner-bool) — a safe
baseline, rmm stays under it (this calibrated layer is needed).

## Evidence-based re-prioritization of the round (2026-06-19, wip start)

Started with the `.gitmodules` layer (the recommendation in "layers" above) → **rejected after a code check**:
the dogfood clone is non-recursive (`scan_repo.sh:17` — `git clone --depth 1 --filter=blob:none`, without
`--recursive`), so boost/ICU/UAPI are committed into the tree **directly**, not as gitlinks; in addition,
the diff/memory mode is structurally blind to submodules (`git ls-tree -r` does not descend into a gitlink, the submodule's
files are not in the parent's tree → `list()` does not see them, nothing to exclude). Corpus measurement: `.gitmodules`
is present in 178/1689 repos, but **populated C++ submodule paths — 3** (a regular clone does not fetch submodules).
`.gitmodules` reaches none of the three dogfood cases → demoted to a separate small
task (will help CI with recursive submodules, no more).

**What actually hits the cases (verified file-by-file):**
- **supercollider** — the container token `external_libraries` (normalized `externallibraries`, NOT in
  `kVendoredDirNames`; only `external`/`externals` are there). Catches both `external_libraries/boost/**`
  and `external_libraries/icu/unicode/*` at once, name-independent, in disk and diff mode. The residual
  `SCDoc/SCDoc.tab.hpp` — bison C++, caught by extending `isGeneratedPath` (`.tab.hpp/.cpp/.cc`;
  currently only `.tab.c`).
- **newsboat** `filter/Parser.h`/`Scanner.h` (Coco/R) — only by the Coco/R/generated **content banner**.
  Matching by basename is NOT allowed (`Parser.h`/`Scanner.h` — entirely your own code). On reclone confirm the banner;
  no banner → residual (do not force).
- **bpftrace** — `posix_types.h` and ~96/118 files under `src/stdlib/include` are caught by the marker
  `Linux-syscall-note`. **Residual — 3 files: `bpf_assembler.h` (held on `copied from`; the marker
  was dropped #131 → returned as FP), `compiler-clang.h`/`errno.h` — a bare single-line `SPDX GPL-2.0`,
  with no distinguishable marker and WITHOUT a nested LICENSE.**
  Verified: in `src/stdlib/` there is no nested LICENSE (only the root one + `tests/.../LICENSE`), and
  a single-line SPDX was rejected as a signal by decision #081. So **nested-LICENSE is useless for bpftrace.**

**Round scope decision: A + B.**
- **A** — the `external_libraries` token.
- **B** — content markers generated/copied via a new `hasGeneratedHeader` in `AuthoredScope` +
  path `.tab.h` in `isGeneratedPath`. The marker set is finalized by measurement (see below): only
  **distinctive** `@generated` / `coco/r` / `linux-syscall-note` (3). The prose-risky
  `generated by` / `imported from` / `do not edit` excluded right away; `copied from` dropped separately
  (#131, see below) — all caught own code.

**Deferred (NOT in this round):** generic nested-LICENSE+code (broad-corpus value, but does not serve
the dogfood goal and carries over-exclude risk — a separate corpus-calibrated round + enumeration of non-project
files, since LICENSE/COPYING do not get into `list()` because of `hasProjectExtension`); the bpftrace residual
`compiler-clang.h`/`errno.h` (needs a `stdlib/include` path token or SPDX-GPL-mismatch — risky);
the `.gitmodules` layer; the transparency line (a single mechanism across all reasons, in a separate commit).
