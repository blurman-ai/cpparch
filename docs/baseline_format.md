# Graph-baseline format

Минимальная спецификация формата `graph-baseline` — снапшота графа зависимостей
проекта, который archcheck кладёт в репо и сравнивает с текущим графом для
DRIFT-правил (см. §«Drift-regression rules» в `docs/architecture-spec.md`).

## Назначение

Baseline — **иммутабельный снимок резолвленных проектных рёбер и узлов** графа.
Это контракт, против которого считаются drift-нарушения. Файл хранится в
репозитории, ревьюится в PR'ах, читается человеком.

## Схема (YAML)

```yaml
format_version: "1"
nodes:
   - "include/foo.h"
   - "include/bar.h"
   - "src/foo.cpp"
edges:
   - [2, 0]
   - [2, 1]
```

Поля:

- `format_version` — строка, текущее значение `"1"`. Строка, а не число, —
  у YAML-чисел свои сюрпризы (`1.0` ≠ `1`, ведущие нули и т.д.).
- `nodes` — массив строк, нормализованных путей узлов графа.
- `edges` — массив пар `[from_idx, to_idx]`, индексы в массив `nodes`.

## Детерминированность

Один и тот же логический граф **обязан** сериализовываться побайтово одинаково,
независимо от порядка вставки узлов и рёбер в `DependencyGraph`. Гарантируется
двумя сортировками:

- `nodes` сортируются лексикографически (по `std::string`).
- `edges` сортируются по паре `(from_idx, to_idx)`, индексы — уже в
  пост-сортировочной нумерации `nodes`.

## Что НЕ входит в baseline

- `external` / `unresolved` / `ambiguous` includes — они нестабильны между
  машинами и сборками.
- Диагностики сканера.
- Производные метрики (`SCC`, `CCD`, `ACD`, `NCCD`, blast radius, hubness) —
  пересчитываются на загрузке.

Baseline хранит только подграф проектных файлов с резолвленными рёбрами.

## Версионирование

Любое изменение схемы (новые поля, переименования, смена семантики) → новое
значение `format_version`. Загрузчик при незнакомой версии возвращает
`BaselineLoadError{Kind::UnknownVersion}` и **не пытается читать дальше**.

## Обработка ошибок

`load_baseline` не бросает исключений наружу. Все ошибки возвращаются как
структурированный `BaselineLoadError`:

| `Kind`             | Когда                                                       |
|--------------------|-------------------------------------------------------------|
| `ParseError`       | невалидный YAML, ryml не смог построить дерево              |
| `UnknownVersion`   | `format_version` отличается от `"1"`                        |
| `MissingField`     | отсутствует `format_version`, `nodes` или `edges`           |
| `MalformedSchema`  | поля есть, но типы/индексы не сходятся (edge вне `nodes`, нечисловой индекс, edge не пара) |

`message` — человекочитаемый текст, безопасный для печати в stderr. `line` —
номер строки в файле, если ryml его сообщил, иначе `0`.
