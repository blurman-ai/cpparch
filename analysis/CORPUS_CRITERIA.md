# Критерии отбора реп для корпуса (constraint-decay research)

**Единый источник истины.** ПЕРЕД тем как клонировать или анализировать любую репу — прогнать
через эти ворота. Не импровизировать, не «на глазок». Если что-то противоречит этому файлу —
прав файл; если файл устарел — переписать файл, а не обходить молча.

> **Окно анализа = последние 24 месяца (политика, 2026-06-12).** При per-commit
> сканировании (graph drift, lateral, boolean, local complexity) реплеить коммиты
> ТОЛЬКО за последние ~2 года, не всю историю — дешевле и свежее. Это окно *анализа*,
> отдельное от критерия *отбора* (1.2, «>300 коммитов с 2025-05-01»). Прежние прогоны
> использовали cutoff 2025-05-01; новые брать since ≈ (today − 24 мес). Параметр в
> скриптах пока хардкод per-script — при следующем прогоне каждого вынести в общий
> `SINCE`/`--since`. Снимок скриптов в `analysis/` отражает то, что реально гонялось.

## TL;DR — ворота перед клоном

Клонируем / берём в анализ репу ТОЛЬКО если выполнены ВСЕ:

- [ ] Язык **C++ (или C)**, primary
- [ ] **> 300 коммитов с 2025-05-01** — точный счёт (`commit_count.py`), НЕ «один коммит в прошлом году, два в этом»
- [ ] **Агентная**: AI-трейлер ИЛИ bot-авторство ≥ 10% ИЛИ медиана длины коммит-сообщения ≥ 100 симв
- [ ] **Не форк**
- [ ] **Не гигант** с большим числом контрибуторов + сильным ревью / CODEOWNERS / branch-protection

Хоть одно ✗ → **НЕ клонировать**. Отметить в реестре как проверенную и идти дальше.

## 1. Жёсткие критерии включения (MUST)

### 1.1 Язык
C++/C, primary language. Поиск: `gh search repos --language cpp|c`.

### 1.2 Скорость: > 300 коммитов с мая 2025 — ГЛАВНЫЙ критерий
**Зачем.** Для наблюдаемой динамики дрифта нужна история. Репа «1 коммит в 2024, 2 в 2025»
структурного долга во времени не покажет — нечего мерить. Это и есть тот случай, который
нельзя пропустить.
**Как считаем БЕЗ клона.** page-count трюк:
`gh api -i "repos/{repo}/commits?since=2025-05-01T00:00:00Z&per_page=1"` → номер последней
страницы (`rel="last"`) == число коммитов в окне. Инструмент: `commit_count.py`.
**Порог:** `commits_since_may > 300`.

### 1.3 Активность в окне
`updated > 2025-05-01` — есть жизнь в окне исследования. Флаг: `--updated ">2025-05-01"`.

### 1.4 Агентный сигнал (иначе не наш кейс)
Хотя бы один из трёх. Инструмент: `screen_corpus.py` (колонки `win_msglen, bot_share, trailers, agentic`).
- **AI-трейлер** — прямой маркер: `co-authored-by: claude|copilot|cursor|codex`, `noreply@anthropic`,
  «generated with claude», «claude code», 🤖.
- **Bot-авторство ≥ 10%** коммитов — прямой маркер: `copilot-swe-agent[bot]`, `google-labs-jules` и пр.
- **Медиана длины коммит-сообщения ≥ 100 симв** в окне — косвенный, trailer-независимый
  (человек ~20–45, AI ~120–1490). **High-confidence ≥ 200** — для шортлиста под дрифт брать этот тир.

### 1.5 Возраст (мягкий, но важен для «до/после»)
- **Идеал:** создана до ~2024 → есть человеческий baseline ДО агентов, видно переключение.
- **Допустимо:** born-agentic (2025), но тогда нет before/after — только конечное состояние долга.

## 2. Жёсткие исключения (MUST NOT)

- **Форки** (`--include-forks=false`).
- **Гиганты** с большим числом контрибуторов + сильным код-ревью + CODEOWNERS / branch-protection.
  У них есть человеческий гард — не наш кейс, только трафик на скачивание жрут.
- **Зеркала / archived / DMCA-пустые / read-only mirrors.**
- Доминирование vendored-кода — само по себе не исключение, НО vendor-filter обязателен (см. §4).

---

## 3. Дисциплина диска и клона (HOW)

- **Не агентные / не > 300 — НЕ клонировать.** Отметить в реестре, идти дальше.
- **git-протокол троттлит** → метаданные (сообщения, счётчики, скорость) брать через **core API**
  (5000/ч, отдельный от git лимит), НЕ клоном.
- Клонировать только то, что реально идёт в drift:
  `git clone --filter=blob:none --shallow-since=2025-05-01` (история не глубже мая).
- Сразу стереть всё, что не C++/C/CMake.
- После скана: **KEEP по умолчанию** — чтобы перепрогнать, если в нашем туле найдём баг.
  Удалять только если дерево > 150 MB или клон битый.

---

## 4. Измерение сигнала (HOW)

- **Vendor-filter ОБЯЗАТЕЛЕН** перед dup/graph: исключать JUCE, Trilinos, Kokkos, SDL, Eigen,
  OpenCV, ggml, llvm, boost и пр. Иначе vendored забивает сигнал (esys/Trilinos 768→15, T5ynth/JUCE 93→3).
- **Метрики дрифта:** cross-module coupling, mutual pairs, cyclic SCC, authored (vendor-filtered) duplication.
- **Time-dynamics:** снапшоты по месяцам/два месяца, `git rev-list -1 --before={date} {orig-ref}`
  (использовать **orig-ref**, НЕ `HEAD` — после checkout HEAD запинен и окно схлопывается).
- **AI-adoption прокси во времени:** медиана длины коммит-сообщения по месяцам (trailer-независимо).

## 5. Воронка с знаменателем (нарратив на выходе)

- **N проверено** (знаменатель, `screen_ledger.tsv`) — каждая репа помечена.
- **M агентских** (по §1.4).
- **K = агентских И > 300 коммитов** (§1.2) → именно эти идут в динамику дрифта.

**Тезис демо** (не отступать): НЕ «ИИ вызвал цикл», а «вот ИИ-разрабатываемый C++ проект без
гарда — и вот структурный долг, который тулза достаёт». Каузацию не доказываем; показываем образцы.
