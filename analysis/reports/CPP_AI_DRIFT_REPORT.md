# Дрейф C++ при агентской разработке — отчёт

**Дата:** 2026-06-06. **Метод:** event-study с контролем, без поддакивания гипотезам.

## Вопрос
Меняет ли переход на агентскую (ИИ) разработку плотность коммитов и структурный долг
(циклы, копипаст) в C++ OSS-репозиториях — и сильнее ли это, чем у обычных (человеческих) реп.

## Метод
- **Агентские:** 100 C++ реп с AI-трейлерами ≥20 из 100 коммитов (`Co-authored-by: Claude/Copilot/...`),
  >300 коммитов с мая 2025, созданы до 2024-06 (есть «до»), не гиганты по размеру.
- **Контроль:** 50 не-агентских C++ реп (человеческая длина сообщений, нет трейлеров), та же эпоха.
- На репу: клон (shallow since 2024, с блобами), `git log` → дата адопции = первый AI-трейлер;
  коммиты/строки/файлы в окне **90 дней ДО vs 90 дней ПОСЛЕ** адопции (для контроля — placebo-дата 2025-09-01);
  archcheck (циклы, копипаст) на снапшоте **до адопции** и **сейчас**.
- **Правило честности:** в статистику только `commits_before ≥ 10` (есть реальное «до»);
  born-agentic (нет «до») — отдельным счётчиком, НЕ в выводах.
- Инструменты: `experiments/ai_repo_run/unified_metrics.py` (сбор), `analyze_unified.py` (разбор).
  Сырьё построчно: `experiments/ai_repo_run/unified_metrics.tsv` (150 реп, все метрики).

## Воронка
150 реп → 44 отсеяны по размеру (асет-гиганты) → 13 born-agentic (нет «до») →
**61 агентская + 31 контроль** надёжных.

## Результаты (difference-in-differences)

| метрика | агентские (n=61) | контроль (n=31) | вывод |
|---|---|---|---|
| коммиты after/before | **1.61×** (резко ≥2×: 37%, не ускор <1.3×: 40%) | 1.00× (12% / 67%) | ИИ резко гонит активность |
| строк на коммит | 190 → **306** (медиана ratio 1.44) | 132 → 130 (0.93) | коммиты крупнее с ИИ |
| файлов на коммит | 4.84 → 4.2 | 3.53 → 3.58 | без разницы |
| доля реп где **циклы выросли** | 3% (2/60) | 6% (2/31) | ИИ НЕ плодит циклы |
| доля реп где **копипаст вырос** | 38% (22/57), Δмед 0 | 32% (10/31), Δмед 0 | почти нет разницы |

## Выводы (объективно)
1. **Активность взлетает с ИИ.** +61% коммитов и +44% строк на коммит после адопции — при
   ПЛОСКОМ контроле (1.00×). Чистый diff-in-diff: эффект ИИ, не общий тренд.
2. **Структурный долг НЕ растёт быстрее, чем у людей.** Циклы плоско (3% vs 6%), копипаст
   почти не отличается (38% vs 32%, медианная дельта 0). В C++ ИИ заливает больше и крупнее
   кода, но циклы/дубли не вспыхивают относительно контроля.

## Что это значит для гипотезы «C++ дисциплинированнее»
Данные **скорее поддерживают** её на коротком горизонте: рост объёма есть, структурного срыва — нет.
**Но утверждать «C++ иммунен к ИИ-дрейфу» нельзя:**
- Окно «после» часто короткое (репы адаптировались в 2025-26), а структурный долг копится медленно —
  «не вспыхнул» может означать «ещё рано».
- archcheck-пороги консервативны (v0.1); часть тонкого дрейфа может не ловиться.
- Выборка — 61 надёжная агентская репа; для длинного хвоста нужен повтор через 6-12 мес.

## Связанные находки
- Кросс-язык (8 языков, 2300 реп): плотность плоская в популяции, рост — агентский ХВОСТ
  (см. `experiments/ai_repo_run/crosslang/`).
- Литература (METR, DORA 2025, GitClear, «Debt Behind the AI Boom»): per-repo velocity ≠
  адопция; всплеск скорости транзиентный, долг копится — согласуется с п.1-2 выше.
